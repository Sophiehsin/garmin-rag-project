# Skill：建立提問筆記集合

**建立日期：** 2026年4月18日  
**目的：** 標準化建立問題與解決方案文件的流程

---

## 適用情境

當用戶要求：
- 「幫我建立提問筆記」
- 「列出我遇到的問題和解決方案」
- 「建立常見問題文件」
- 類似的問題集合整理需求

---

## 執行流程

### 第 1 步：蒐集問題清單

從對話記錄中識別所有用戶提出的問題，包括：
- 明確的「疑問」和「問題」
- 遇到的錯誤訊息
- 對工具/流程的不理解
- 尋求解決方案的地方

### 第 2 步：篩選問題

根據以下規則篩選問題：

**排除以下問題：**
- 純粹由拼寫錯誤引起且無觀念相關的問題
  - 例如：資料庫名稱打錯 `vsctordb` vs `vectordb`（純粹拼寫）
  - 除非該問題涉及「如何避免拼寫錯誤」或「驗證資訊正確性的方法」等觀念

**保留以下問題：**
- 流程和工具使用方面的疑問
- 概念理解不清楚的地方
- 錯誤排查方法
- 最佳實踐建議

### 第 3 步：組織問題格式

每個問題遵循以下固定格式：

```markdown
## 疑問 X：[問題標題]

**問題：**
[用戶實際遇到的情況描述]

**根本問題：**
[問題的根本原因是什麼]

**解決方案：**
[清楚的解決步驟或方法，可包含程式碼範例]

**其他小筆記：**
[相關的額外資訊、提示、最佳實踐等]
```

### 第 4 步：編寫文件

**檔案名稱規則：**
- 保留現有檔名或建立新檔案
- 通常為項目相關的 `*_QUESTIONS.md` 或 `*_NOTES.md`

**文件結構：**
1. 標題 + 建立日期 + 項目描述
2. 問題列表（按照 1, 2, 3... 編號）
3. 快速參考區段（可選，包含常用命令、連線資訊等）
4. 最後更新時間

**格式要求：**
- 無任何 emoji 或表情符號
- 使用 Markdown 標準語法
- 程式碼使用三個反引號 ``` 包住
- 表格使用 Markdown 表格語法
- 清晰的層級結構（#, ##, ###）

### 第 5 步：生成檔案

使用 `create_file` 工具生成 Markdown 檔案到專案根目錄

---

## 格式範本

```markdown
# [項目名稱] - [主題] 常見問題與解決方案

**建立日期：** YYYY年M月D日
**項目：** [簡短項目描述]

---

## 疑問 1：[問題標題]

**問題：**
[具體描述用戶遇到的問題]

**根本問題：**
[分析問題的本質]

**解決方案：**
[提供清晰的解決步驟]

**其他小筆記：**
[補充資訊]

---

## 疑問 2：[問題標題]

[遵循相同格式]

---

## 快速參考

### 常用命令
[列出重要命令]

### 連線資訊 / 配置
[列出關鍵資訊]

### 相關檔案
[列出重要檔案位置]

---

**最後更新時間：** YYYY年M月D日
```

---

## 品質檢查清單

執行完成後檢查：

- [ ] 所有問題都有「問題」、「根本問題」、「解決方案」、「其他小筆記」四個部分
- [ ] 沒有任何 emoji 或表情符號
- [ ] 程式碼範例正確且完整
- [ ] 排除了純拼寫錯誤但無觀念相關的問題
- [ ] 問題編號連續（1, 2, 3...）
- [ ] 有快速參考區段（如適用）
- [ ] 檔案位置正確（通常在專案根目錄）
- [ ] 有建立日期和最後更新時間

---

## 執行範例

**使用者命令：**
「幫我建立提問筆記」

**執行步驟：**
1. 從對話記錄中提取所有問題
2. 篩選排除純拼寫錯誤
3. 按照格式組織每個問題
4. 編寫完整的 Markdown 檔案
5. 使用 `create_file` 生成檔案

**產出：**
- Markdown 檔案（如 DATABASE_SETUP_QUESTIONS.md）
- 包含 10+ 個結構化問題
- 完整的快速參考資訊

---

## 注意事項

1. **問題數量：** 根據實際情況決定，不強制特定數量
2. **順序：** 按照邏輯順序組織（不一定按對話順序）
3. **相關性：** 確保所有問題都與主題相關
4. **完整性：** 每個解決方案都應該可以獨立參考
5. **可讀性：** 優先使用清晰的自然語言，避免過度技術性

---

## 與其他任務的區別

**不適用此 Skill 的情況：**
- 單純的代碼實現任務
- 實時問題排查（直接修復而非文檔化）
- 功能開發和測試
- 即時的一次性提問

**適用此 Skill 的情況：**
- 整理過去遇到的問題
- 建立知識庫
- 文檔化學習過程
- 標準化常見問題集合
